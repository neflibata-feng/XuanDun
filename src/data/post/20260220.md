## 写在前面
这周我们开始做「每周 GitHub」系列。

如果你最近也在折腾智能体，可能会遇到一个尴尬问题：

“我已经有 ChatGPT / Claude 了，但它们都在网页里；我想要一个能在我的常用聊天软件里随叫随到、还能接工具、还能长期常驻的‘个人助理’。”

今天推荐的项目就是为这个场景而生：**OpenClaw**。

## 1 分钟看懂 OpenClaw

一句话：OpenClaw 是一个**跑在你自己设备上**的个人 AI 助手，它会出现在你常用的聊天渠道里（例如 WhatsApp、Telegram、Slack、Discord、Google Chat、Signal、iMessage、Microsoft Teams、WebChat 等），并提供一个本地优先的「Gateway 控制平面」来管理会话、工具、通道与事件。

你可以理解为：它不是“又一个聊天机器人”，而是一个**可部署、可常驻、可接入多渠道**的个人助手平台。

## 为什么我觉得它“好玩”（也是它最值钱的点）

很多开源智能体看起来很酷，但要么太像框架、要么太像 Demo。

OpenClaw 更像一套“能落地使用”的产品化组合：

- **本地优先 Gateway**：统一控制平面，核心服务常驻
- **多通道收件箱**：不改习惯，在你常用的 IM 里直接对话
- **多 Agent 路由**：不同账号/群聊/渠道可以隔离到不同工作区与会话
- **可选语音 + Live Canvas**：让它从“聊天”走向“长期协作”

## 适合谁 / 不太适合谁

适合你如果：

- 你想要“个人助手”而不是“网页对话框”
- 你希望在多个聊天工具里统一体验（而不是每个平台单独接一套 Bot）
- 你愿意稍微动手部署，并且对“常驻服务 + 配置文件”不排斥

不太适合你如果：

- 你只想要开箱即用的 SaaS，不想碰命令行
- 你完全不想处理授权、配对、渠道接入这些工程问题

## 5 分钟上手（推荐路线）

运行环境：**Node.js ≥ 22**。

官方推荐直接跑引导向导（Wizard）。它会一步步带你把 Gateway、Workspace、Channels、Skills 配好。

```bash
npm install -g openclaw@latest
# 或：pnpm add -g openclaw@latest

openclaw onboard --install-daemon
```

向导会把 Gateway 作为后台服务安装（macOS 对应 launchd，Linux 对应 systemd）。

Windows 用户注意：官方建议通过 **WSL2** 来跑（体验和兼容性更稳）。

## 快速体验（TL;DR）

```bash
# 一键向导（推荐）
openclaw onboard --install-daemon

# 启动网关（控制平面）
openclaw gateway --port 18789 --verbose

# 直接对助手提问
openclaw agent --message "帮我生成一份发版检查清单" --thinking high
```

升级与排障：更新指南 https://docs.openclaw.ai/install/updating ，建议顺手跑一下 `openclaw doctor`。

## 模型与授权怎么选（轻量版）

它支持多模型、多授权方式（OAuth / API Key），并且支持在不同配置之间切换与降级：

- 模型配置：https://docs.openclaw.ai/concepts/models
- 故障切换：https://docs.openclaw.ai/concepts/model-failover

项目文档里对“长上下文 / 抗提示注入”等有偏好建议，但你完全可以按预算和可用性来选。

## 最后一个必须强调：安全默认值

它会接入真实消息渠道，所以请把陌生私信当作**不可信输入**。

默认常见策略是“DM 配对”：

- 陌生人私信会收到配对码，机器人不会直接处理内容
- 你需要手动批准：`openclaw pairing approve <channel> <code>`
- 如果你真要开放陌生来源，请显式开启 `dmPolicy="open"` 并配 allowlist

安全文档：https://docs.openclaw.ai/gateway/security

## 我会怎么用它

如果你是单人效率党：把它接入 1-2 个你最常用的渠道（比如 Telegram/Slack），先做“收件箱 + 代办 + 总结”，再慢慢加工具。

如果你是团队：建议先把“群聊”与“主会话”隔离，把默认策略跑通后再逐步开放能力。
