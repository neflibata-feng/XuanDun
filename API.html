多数据源融合的互联网个人贷款及全流程风控系统

1.文档信息

  • 项目名称：多数据源融合的互联网个人贷款及全流程风控系统API文档。
  • 版本：v1.0.0
  • 描述：本文档定义了个人贷款系统前后端交互的接口规范，所有接口遵循RESTful设计原则。

2.通用约定

基础信息

  • Base URL：https://www.xxxxxx.com/v1
  • 数据格式：请求与响应主体均使用 application/json。
  • 字符编码：UTF-8

受保护的接口需在 HTTP Header中携带通过OAuth 2.0获取的访问令牌（Token）

 Authorization: Bearer <token>

返回码

  • 若请求失败，API会返回如下JSON格式的返回码及说明
{
    "error":{
        "code": "****",
        "message": "*****"
    }
}

标准HTTP状态码

状态码 含义 说明
200 OK 请求成功
201 Created 资源创建成功（如注册等）
400 Bad Request 请求参数错误
401 Unauthorized 未认证或Token无效
403 Forbidden 权限不足
404 Not Found 请求的资源不存在
500 Internal Server Error 服务器内部错误

分页与列表查询

  • 对于返回列表的接口，支持分页查询
请求参数 类型 含义
page int 页码（从1开始）
size int 每页大小（默认为10） 
{
    "code": 200,
    "message": "Success",
    "data":{
        "content": [] ,//当前返回的数据列表
        "totalPages": 5,
        "totalElements": 15
        "size": 10,
        "page": 1
    }
}
返回参数 类型 含义
totalPages int 总页数
totalElements int 总记录数

3.接口详述

3.1 用户注册登录

3.1.1 用户注册

  • 功能说明：用户输入手机号和验证码注册账号
  • 功能需求：FR-01
  • Endpoint: POST https://www.xxxxxx.com/v1/users (Basse URL以下皆省略，即省略“https://www.xxxxxx.com/v1”)
  • 请求体：
{
    "mobile": "13607385337",
    "password": "sjm123456",
    "smsCode": "554442"
}
  • 请求参数
关键字 类型 含义
mobile String 用户手机号码
password String 账号密码
smsCode String 6位验证码
  • 成功响应
{
    "code": 201,
    "message": "User registered successfully",
    "data": {
        "userID": "10001"
    },
    "timestamp": 1719832845000
}
  • 响应参数
    关键字 类型 含义
    code int 返回码
    message string 响应说明
    userID string 系统为用户定义的账户ID
    timestamp long 时间戳

3.1.2 用户登录

  • 功能需求：FR-01

}
  • 请求参数
关键字 类型 含义

{

    "message": "Login successfully",
    "data": {
        "accessToken": "...........",
        "tokenType": "Bearer"
    },
    "timestamp":
}
  • 响应参数
关键字 类型 含义
accessToken string 访问令牌
tokenType string 令牌类型说明

3.5 用户还款

3.5.1 用户查询还款计划

  • 功能说明：获取用户某一笔贷款的还款计划。
  • 功能需求：FR-06
  • Endpoint: GET /loan/repayment/{applicationId}/plans
  • Header: Authorization: bearer <token>
  • 成功响应
{
  "code": 200,
  "message": "Success",
  "data": {
    "applicationId": "APP202411080001",
    "totalPeriods": 12,
    "repaidPeriods": 3,
    "repaymentPlans": [
      {
        "periodNumber": 4,
        "dueDate": 1722504000000, 
        "principalAmount": 833.33, 
        "interestAmount": 100.00,  
        "totalAmount": 933.33,   
        "status": "DUE",          // 状态: UPCOMING(未到期), DUE(待还款), OVERDUE(逾期), REPAID(已还)
        "repaidTime": null        // 若已还，则为还款时间
      },
      // ... 其他期数计划
    ]
  },
  "timestamp": 
}
  • 响应参数
关键字 类型 含义
totalPeriods int 还款总期数
repaidPeriods int 已还期数
periodNumber int 当前期数
dueDate long 截止时间戳
principalAmount decimal 本期应还本金
interestAmount decimal 本期应还利息
totalAmount decimal 本期应还金额
repaidTime string 还款时间

3.5.2 发起还款请求

  • 功能说明：用户为某一笔贷款的特定期数发起还款。

  • 功能需求：FR-06

  • Endpoint: POST /loan/repayment/{applicationId}

  • Headers: Authorization: bearer <token>

  • 请求体

{
  "applicationId": "APP202411080001",
  "repayPeriod": 4, // 偿还第4期。如为0，则表示提前结清所有未还期数。
  "paymentMethod": "BANK_CARD" // 支付方式：银行卡、其他余额等
}
  • 成功响应
{
  "code": 200,
  "message": "Repayment request accepted, redirecting to payment...",
  "data": {
    "repaymentId": "REP202411080001", // 本次还款请求的唯一ID
    "paymentUrl": "https://payment.com/pay/xxx" // 支付网关地址（如需跳转）
  },
  "timestamp": 
}

3.6 执行审批

3.6.1 系统自动审批

  • 功能说明：由系统内部触发的流程，当用户提交贷款申请后，系统自动调用风控模型进行决策。不直接对前端开放。

  • 功能需求：FR-04

  • Endpoint: POST /internal/risk/auto-approval

  • 触发条件：用户提交贷款申请（POST /loan/applications）,由系统自动调用。

  • 请求体

{
  "applicationId": "APP202411080001",
  "userId": 10001,
  "applyAmount": 10000.00,
  "loanTerm": 6,
  "userData": {
    "age": 30,
    "income": 15000.00,
    // ... 其他风控所需数据
    }
}
  • 请求参数
关键字 类型 含义
applyAmount decimal 申请金额
loanTerm int 贷款期数
age int 年龄
income decimal 收入

  • 处理逻辑:
    系统整合内外部数据，调用反欺诈和信用评分模型。
    根据预置的业务规则（如额度限制、征信准入规则）进行决策。
    更新贷款申请的状态。

  • 决策结果与状态映射:
    自动通过 -> 申请状态更新为  APPROVED 
    自动拒绝 -> 申请状态更新为  REJECTED 
    转人工审核 -> 申请状态更新为  PENDING_MANUAL_REVIEW

3.6.3 提交人工审批结果

  • 功能说明：管理员对需要人工审批的申请进行审批，提交结果及理由

  • 功能需求：FR-08

  • Endpoint: POST /admin/loan/applications/{applicationId}/manual-approval

  • Headers: Authorization: bearer <token>

  • 请求体

{
  "decision": "APPROVED", // 审批决定：APPROVED（通过） / REJECTED（拒绝）
  "approvalAmount": 10000.00, // 审批金额（可<=申请金额）
  "approvalTerm": 6,
  "remarks": "经过人工复核用户银行流水，收入稳定，通过。" // 审批备注/理由
}
  • 请求参数
关键字 类型 含义
decision string 审批结果
approvalAmount decimal 审批金额
approvalTerm int 审批期限
remarks string 审批备注
  • 成功响应
{
  "code": 200,
  "message": "Manual approval submitted successfully",
  "data": {
    "applicationId": "APP202411080001",
    "Status": "APPROVED", // 申请状态更新为已批准
    "approver": "admin_user01",
    "approvalTime": 1719833000000
  },
  "timestamp": 

}
  • 响应参数
关键字 类型 含义
approver string 审批员
approvalTime long 审批时间戳
  • 业务规则:
    只有状态为  PENDING_MANUAL_REVIEW  的申请才能进行人工审批操作。
    操作完成后，申请状态根据决策更新为  APPROVED 或 REJIECTED 。
    审批理由（ remarks ）为必填项，以满足审计要求。
